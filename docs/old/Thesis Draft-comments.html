faster
more specifically:
- GC has better throughput; in an hour of runtime, GC will spend less time on memory management, and produce more useful calculations. however, itll periodically pause.
- RC has better latency, because it doesn't periodically pause. we can guarantee "each request is handled within 1ms" with RC, but not with RC.

(by "latency" i dont mean over the network, i mean time to do a calculation within the machine)

for example (making up numbers):
- an RC server can handle 1000 QPS guaranteeing <5ms per request (nice)
- a GC server can handle 1500 QPS but only guaranteeing <50ms per request (ew)
- hopefully, HGM can give us 1600 QPS with <2ms per request
